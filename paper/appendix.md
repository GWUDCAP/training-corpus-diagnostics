# Appendix: Release Tables and Reproducibility Notes

This appendix summarizes the public release artifacts for the lens-effects corpus paper. Machine-readable sources are included next to each table so the numbers can be checked or regenerated.

## A. Corpus Panel

Source: `data/corpus_manifest.json`.

All retained comparisons use 2048 chunks per corpus, chunks of up to 768 characters, and at most two chunks per source document. Under the retained builder, chunks shorter than `max(64, chunk_chars // 4)` are discarded, which is 192 characters under the retained setting. The retained panel contains six pretraining corpora and eight post-training corpora.

The retained panel is generated by randomized reservoir sampling. For streaming sources, the builder scans up to 50,000 nonempty source documents per corpus, maintains a 12,000-document reservoir, and chunks sampled documents until 2048 chunks are retained. Source adapters preserve the same retained-sample contract while recording source-specific references such as dataset row positions or shard references. Retained samples include `chunk_metadata.json` and `chunks_provenance.jsonl` sidecars recording sampling mode, seed, source position, source reference, source-text hash, chunk hash, character spans, and document sample rank.

## D. Post-Training Structure

Source: `results/final_matrix/targeted_substructure_summary.json`.

Representative post-training/post-training comparisons show that the post-training side is internally structured rather than a single residual class.

| OASST1 vs UltraFeedback | 0.0320 | 0.7302 | 0.0619 | Mild assistant/preference neighborhood |

Semantic cluster JS is reported in natural-log units and is bounded above by `ln(2) = 0.6931`. Values near that ceiling indicate nearly disjoint cluster occupancy under the retained partition.

## E. Directed Mixtures

Source: `results/final_matrix/mixtures/targeted_mixture_panel.json`.

The mixture panel covers all 56 directed post-training trajectories over alpha values 0.0, 0.1, 0.25, 0.5, 0.75, and 1.0. Near-sibling HH chosen/rejected trajectories are nearly flat. Stronger trajectories involving Nemotron-SFT-Math or Magicoder OSS Instruct reach much larger endpoint semantic and lexical separations. Some semantic endpoints approach the `ln(2)` cluster-JS ceiling; for those trajectories, the interior path remains informative, but the near-saturated endpoint values should not be used for fine ranking by cluster JS alone.

## F. Stability Checks

Sources: `stability/bootstrap_canonical_summary.json`, `stability/leave_one_out_summary.json`, `stability/same_corpus_split_baselines.json`, and `stability/semantic_k_stability.json`.

Bootstrap resampling preserved the expected pretraining/post-training ordering in 200/200 replicates for all six retained summaries. The semantic bootstrap summary uses the retained canonical pairwise partitions while resampling chunk assignments within each corpus; the nonsemantic summaries are recomputed on the resampled chunks.

Leave-one-corpus-out checks preserved all six orderings under every omission. The retained panel has 15 within-pretraining pairs and 48 pretraining/post-training pairs; each pretraining omission leaves 10 and 40 such pairs, and each post-training omission leaves 15 and 42.

Same-corpus split baselines were substantially below the pretraining/post-training scale for divergence-style summaries. Pooled same-corpus lexical JS was 0.0294 and pooled same-corpus semantic cluster JS was 0.0055.

The retained extraction uses randomized reservoir sampling with provenance sidecars. The randomized retained panel preserved all six orderings over the full fourteen-corpus panel.

The GTE c256 sweep truncates more aggressively than the retained c768 setting and still preserves the semantic ordering, with within-pretraining cluster JS 0.0593 versus pretraining/post-training cluster JS 0.3400.

Semantic cluster-count stability preserved the semantic ordering for all tested `k` values:

## G. Continuous Embedding Ceiling Sensitivity

Source: `results/final_matrix/continuous_embedding_sensitivity.json`.

Cluster JS is intentionally discrete: it measures occupancy shifts over a retained KMeans partition. For near-ceiling cluster-JS pairs, the release includes a non-binned continuous embedding-distance sensitivity check using row-normalized cached embeddings and fixed random projections. This is a ceiling-sensitivity companion, not a bootstrapped headline family.

| Pair class | Count | Centroid cosine | Centroid L2 | Sliced W1 | Sliced W2 |
|---|---:|---:|---:|---:|---:|
| Within-pretraining | 15 | 0.0160 | 0.0781 | 0.0026 | 0.0033 |
| Pretraining/post-training | 48 | 0.1587 | 0.2783 | 0.0084 | 0.0105 |
| Post-training/post-training | 28 | 0.2262 | 0.3461 | 0.0105 | 0.0130 |

Among near-ceiling cluster-JS pairs, the largest sliced-W1 distances involve Nemotron-SFT-Math against BeaverTails, HH chosen/rejected, Dolly, and several broad corpora. The check therefore preserves the qualitative strong-pole interpretation while supplying a continuous ranking where cluster JS is near saturation.

## H. Reproducibility

Source map: `stability/repo_map.md`.

Quick artifact verification:

```bash
python scripts/verify_release_artifacts.py
```

Final matrix recomputation:

```bash
python scripts/run_corpus_matrix.py \
  --manifest data/corpus_manifest.json \
  --out-dir recomputed
```

Directed mixture-panel recomputation:

```bash
python scripts/run_mixture_panel.py \
  --manifest data/corpus_manifest.json \
  --out-dir results/final_matrix/mixtures
```

Fresh source-level regeneration:

```bash
python scripts/build_corpus_artifacts.py build-chunks \
  --sources configs/paper_panel_sources_randomized_2048.json \
  --out-root reproduced_data

python scripts/build_corpus_artifacts.py build-embeddings \
  --manifest reproduced_data/corpus_manifest.generated.json \
  --out-root reproduced_data
```

To regenerate a randomized retained panel, run the chunk builder with the randomized source recipe before embedding and matrix recomputation. The retained recipe uses `sampling_mode: reservoir`, `sample_seed: 20260505`, a 50,000-document scan budget, a 12,000-document reservoir, and 2048 retained chunks per corpus.

Stability recomputation:

```bash
python scripts/run_bootstrap_hardening.py \
  --manifest data/corpus_manifest.json \
  --scorecard-dir results/final_matrix/recomputed \
  --out-dir stability \
  --bootstrap-semantic-mode fixed
```

The package supports reproduction of the released corpus statistics, regeneration of fresh source-level samples, and substitution of new corpus manifests. It does not establish downstream causal effects or a universal taxonomy of all corpora.

The retained chunks and cached embeddings are the paper-facing sample and support artifact-level recomputation. The retained sample includes provenance sidecars for chunk hashes, source references, source positions when available, source-text hashes, character spans, and sampling parameters. The release also includes `scripts/build_corpus_artifacts.py`, which regenerates chunks from the public source recipe or local JSONL/text inputs, writes provenance sidecars, builds GTE embeddings, and emits a manifest compatible with the matrix runner. Reproducing the retained sample exactly requires the retained source recipe, dataset revisions, streaming behavior, shuffle implementation, and sampling parameters.
